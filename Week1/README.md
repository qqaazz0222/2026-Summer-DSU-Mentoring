## 1. 웹은 어떻게 동작하나

```
[브라우저(클라이언트)]  --- 요청(Request) --->  [서버]
[브라우저(클라이언트)]  <--- 응답(Response) ---  [서버]
```

- 주소창에 URL을 입력하면 브라우저가 서버에 "이 페이지 주세요"라고 요청합니다
- 서버가 HTML 파일(+CSS/JS/이미지 등)을 응답으로 보냅니다
- 브라우저가 HTML을 해석해서 화면에 그립니다 (렌더링)
- 지금 Week1~2는 서버 없이 로컬 HTML 파일을 브라우저로 직접 열어서 확인합니다. 서버 개념은 Week4~5에서 실제로 다룹니다

### URL 구조
```
https://example.com/products?id=3
   │        │           │      │
프로토콜   도메인       경로   쿼리(파라미터)
```

## 2. HTML 문서 기본 구조

```html
<!DOCTYPE html>
<html lang="ko">
  <head>
    <meta charset="UTF-8" />
    <title>페이지 제목</title>
  </head>
  <body>
    <!-- 화면에 보이는 내용은 전부 여기 -->
  </body>
</html>
```

- `<!DOCTYPE html>` : HTML5 문서라는 선언, 항상 맨 위 1줄
- `<html>` : 문서 전체를 감싸는 최상위 태그
- `<head>` : 화면에 안 보이는 메타정보 (제목, 인코딩 등)
- `<body>` : 실제 화면에 보이는 내용

## 3. 기본 태그 정리

| 태그 | 용도 |
|---|---|
| `<h1>` ~ `<h6>` | 제목 (숫자 작을수록 큰 제목, h1은 페이지당 보통 1번) |
| `<p>` | 문단 |
| `<a href="...">` | 링크 |
| `<img src="..." alt="...">` | 이미지 (alt는 대체 텍스트, 접근성/SEO에 중요) |
| `<ul>` / `<ol>` + `<li>` | 순서 없는/있는 목록 |
| `<div>` | 의미 없는 블록 구획 (레이아웃용) |
| `<span>` | 의미 없는 인라인 구획 (텍스트 일부 스타일 줄 때) |
| `<form>` | 입력 양식 묶음 |
| `<input>` | 입력창 (type="text", "email", "checkbox" 등) |
| `<button>` | 버튼 |

`div`/`span`은 아무 의미 없는 상자입니다. 레이아웃을 잡을 때 씁니다 (Week2 CSS에서 본격 활용합니다).

`SOURCE/01_basic_tags.html` 파일에서 실제로 확인.

## 4. 시맨틱 태그

`div`만 쓰면 코드를 읽을 때 이게 뭔지 알기 어렵습니다. 의미 있는 태그로 구역을 나누면 가독성/접근성이 좋아집니다.

| 태그 | 의미 |
|---|---|
| `<header>` | 페이지/섹션 상단 (로고, 제목 등) |
| `<nav>` | 내비게이션(메뉴) 영역 |
| `<main>` | 페이지 핵심 콘텐츠 (문서당 1번) |
| `<section>` | 주제별 구획 |
| `<article>` | 독립적으로 의미 있는 콘텐츠 (게시글 등) |
| `<footer>` | 페이지/섹션 하단 (저작권, 연락처 등) |
